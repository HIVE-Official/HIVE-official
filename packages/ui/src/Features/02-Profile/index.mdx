# 02-Profile

**User identity and profile management**

## Purpose

Profile is the user's campus identity. It shows who they are, what they're interested in, who they're connected with, and what spaces they're part of.

## Components in this Feature

- **Profile Bento Grid** - Adaptive profile layout system
- **Profile Identity Widget** - User's basic info and status
- **Profile Activity Widget** - Recent activity timeline
- **Profile Connections Widget** - Friend connections list
- **Profile Spaces Widget** - User's communities
- **Profile Completion Card** - Profile completion status and tips
- **Profile View Layout** - Full profile page template
- **Avatar Upload with Crop** - Profile picture management
- **Presence Indicator** - Online/offline status

## Key Flows

1. **View Own Profile** - See your profile as you see it
2. **View Other Profile** - See someone else's public profile
3. **Edit Profile** - Update your information
4. **Complete Profile** - Fill in missing sections

## Design Principles

- **Authentic** - Real campus identity, not LinkedIn
- **Social Context** - Show mutual connections and spaces
- **Privacy-First** - Control what others see
- **Completion-Driven** - Encourage profile completion

## Profile Completion

Target: 100% profile completion
- ✅ Avatar (required)
- ✅ Bio (optional but encouraged)
- ✅ Interests (minimum 3)
- ✅ At least 1 space joined
- ✅ At least 1 connection

## Success Metrics

- Profile completion rate: > 90%
- Average profile views per user: 5-10/week
- Connection requests sent: 3-5/week

## Design Principles

### Effortless Competence
Make users feel capable from the start:
- âœ… Pre-filled when possible
- âœ… Clear, simple language
- âœ… Visual progress indicators
- âœ… Immediate positive feedback

### 70% Completion Target
Optimize for finishing, not perfection:
- Students see value before 100% complete
- Can skip non-essential steps
- Return later to complete profile
- No "required field" anxiety

### Social Proof
Show community from the start:
- "Join 5,000 UB students"
- Preview spaces they can join
- Show activity happening now
- Create FOMO (fear of missing out)

### Relief Amplifier
Frame as "someone needs you":
- "Your classmates are waiting"
- "People want to connect with you"
- "You're needed in [Space Name]"
- Creates urgency through community need

## Behavioral Psychology Patterns

### Variable Reward Scheduling
- Surprise elements during onboarding
- Unexpected community welcomes
- Random positive reinforcement

### Completion Psychology
- Progress bars calibrated to 70%
- Micro-celebrations at each step
- "Almost there!" messaging

### Social Proof
- Real student count (not fake)
- Live activity indicators
- "X people joined today"

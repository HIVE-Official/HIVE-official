# 07-Notifications

**Alert and notification system**

## Purpose

Keep users informed about important campus activity: space updates, connection requests, mentions, and ritual participation.

## Components in this Feature

- **Notification System** - Complete notification infrastructure
- **Notification Dropdown** - Notification center in header
- **Notification Toast Manager** - Ephemeral toast notifications
- **Notification Bell** - Icon with unread count
- **Notification Item** - Individual notification card

## Notification Types

1. **Social** - Connection requests, mentions, messages
2. **Spaces** - New posts, event reminders, admin updates
3. **Rituals** - Campaign updates, participation milestones
4. **System** - Account updates, policy changes

## Design Principles

- **Actionable** - Every notification has a clear action
- **Timely** - Delivered within seconds
- **Relevant** - Smart filtering to avoid noise
- **Dismissible** - Easy to mark as read or dismiss

# HIVE Platform Layout Architecture

## Core Layout Philosophy

### The Anti-Pattern Pattern
While everyone else does centered, symmetrical layouts, HIVE uses **intentional asymmetry** inspired by:
- **Discord**: Persistent sidebar navigation
- **TikTok**: Full-screen immersion with overlay UI
- **Spotify**: Contextual panels that slide and stack
- **BeReal**: Time-sensitive content prioritization

### Mobile-First Grid System
```css
/* HIVE Custom Grid - Not your typical 12-column */
.hive-grid {
  --mobile: 4 columns (swipe-based)
  --tablet: 8 columns (hybrid touch/mouse)
  --desktop: 16 columns (asymmetric splits: 3-10-3, 5-11, etc.)
}
```

## Layout Principles

### 1. Never Center Everything
- Left-aligned content reads faster
- Asymmetric layouts create visual interest
- Offset grids guide the eye naturally

### 2. Z-axis Thinking
```
Layer 0: Background (black)
Layer 1: Content cards
Layer 2: Navigation
Layer 3: Floating elements (FABs)
Layer 4: Modals/overlays
Layer 5: Toasts/notifications
```

### 3. Thumb-Friendly Mobile
- Bottom navigation for mobile
- FABs in thumb reach zone
- Swipe gestures for primary actions

### 4. Information Density
- **Mobile**: Low density, large touch targets
- **Tablet**: Medium density, mixed interaction
- **Desktop**: High density, hover states

### 5. Loading States
- Skeleton screens match final layout
- Progressive content loading
- Maintain layout stability (no jumps)

---

## Implementation Priority

1. **Feed** - Most used, sets the tone
2. **Spaces** - Discovery is critical
3. **Profile** - Identity must be clear
4. **Messaging** - Real-time communication
5. **Events** - Calendar integration
6. **Rituals** - Engagement driver
7. **HiveLab** - Power user feature
